## 📚 Contribution Guidelines

### The HANERMA Way
1. **Mathematical rigor** in every contribution
2. **Zero-fluff mandate** - no unnecessary complexity
3. **Production readiness** - every PR should be production-quality
4. **Community first** - prioritize contributor success
5. **Documentation driven** - every feature needs clear docs

### Code Standards
- **Rust safety**: Memory ownership, no unsafe blocks
- **Python clarity**: Type hints, docstrings, error handling
- **Test coverage**: Every feature needs tests
- **Performance**: No regressions, profile optimizations

### Culture Values
- **Mathematical certainty** over heuristic approaches
- **Formal verification** over empirical testing
- **Systems programming** over manual memory management
- **Community growth** over gatekeeping
- **Collaboration** over competition
